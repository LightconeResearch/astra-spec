# About ASTRA

**ASTRA** stands for *Agentic Schema for Transparent Research Analysis*. It is a declarative format for describing scientific analyses in a way that is precise enough to drive automated execution and transparent enough to be audited, reproduced, and revisited.

## The two repositories

ASTRA is split across two packages with a clean boundary at the agentic layer.

| Package | Repository | Purpose |
|---------|-----------|---------|
| [`astra-spec`](https://pypi.org/project/astra-spec/) | [`LightconeResearch/astra-spec`](https://github.com/LightconeResearch/astra-spec) | LinkML schema, generated bindings, and this documentation site. |
| [`astra-tools`](https://pypi.org/project/astra-tools/) | [`LightconeResearch/ASTRA`](https://github.com/LightconeResearch/ASTRA) | Python CLI and SDK: validation, paper cache, evidence verification, helpers. |

Both packages share the `astra.*` Python namespace via [PEP 420](https://peps.python.org/pep-0420/) implicit namespace packages — there is intentionally no `src/astra/__init__.py` in either repo.

`astra-spec` is the authoritative source for the specification. Schema definitions, valid field names, and semantic rules originate there. `astra-tools` consumes the schema and provides the human and machine surface for working with it.

## Out of scope

ASTRA is intentionally agnostic to the agentic and execution layers. The specification and tooling do not:

- prescribe how to run an analysis — any agent, workflow engine, notebook, or human can consume an ASTRA spec;
- store data — ASTRA references data sources but does not persist them;
- generate code — the spec is not a template, and the tooling does not emit implementations;
- render results — visualisation and reporting are handled by downstream tools.

## Schema artefacts

The LinkML schema is published in several formats, all derived from the source files under [`src/astra/schema/`](https://github.com/LightconeResearch/astra-spec/tree/main/src/astra/schema):

| Format | Path |
|--------|------|
| LinkML YAML (source) | [`schema/astra.yaml`](schema/astra.yaml) |
| JSON Schema | [`schema/astra.schema.json`](schema/astra.schema.json) |
| JSON-LD context | [`schema/astra.context.jsonld`](schema/astra.context.jsonld) |

Additional language bindings (Python dataclasses, Pydantic, TypeScript, Java, OWL) are generated by `just gen-project`; see the [project layout on GitHub](https://github.com/LightconeResearch/astra-spec/tree/main/project).
